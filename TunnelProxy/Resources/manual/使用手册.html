Tunnel Proxy — 使用手册

一份面向普通用户的 Tunnel Proxy（macOS 菜单栏应用）友好指南。

Tunnel Proxy 是什么？

Tunnel Proxy 是一款 macOS 应用，它通过你自己的 SSH 服务器，用一条安全加密的连接来转发你的网络流量。它有一个供日常使用的主窗口，以及一个用于快速操作的菜单栏盾牌图标（位于屏幕右上角那排图标中）。

你可以用它来：

  • 通过位于其他地区的服务器访问互联网；
  • 在不可信的网络（如公共 Wi-Fi）上保护你的流量隐私；
  • 让网站和应用看到你服务器的 IP 地址，而不是你 Mac 的 IP。

它所需的一切都已内置——无需再安装任何其他软件。你只需要知道如何连接一台 SSH 服务器（它的地址，以及你的登录方式）。

工作原理

你的应用把流量发送到 Mac 上运行的本机代理，Tunnel Proxy 通过加密隧道把这些流量送到你的 SSH 服务器，再由服务器转发到互联网。你访问的网站看到的是服务器的 IP 地址，而你的 Mac 与服务器之间的一切都经过加密。

安装应用

  1. 打开你收到的 TunnelProxy.dmg 文件。
  2. 把 Tunnel Proxy 图标拖到 应用程序（Applications） 文件夹上。
  3. 打开 应用程序，双击 Tunnel Proxy。
    • 首次打开时，macOS 可能会询问「确定要打开吗？」——点击 打开。（如果被系统拦截，请前往 系统设置 → 隐私与安全性，向下滚动，点击 仍要打开。）
  4. 主窗口会打开，同时菜单栏中会出现一个盾牌图标。你可以从任一处操作应用。

提示： 没看到盾牌？可能是菜单栏太满了。按住 ⌘ 拖动一些菜单栏图标腾出空间，或确认应用是否已在运行。你随时可以通过菜单栏弹出面板中的 打开应用 按钮重新打开主窗口。

主窗口

主窗口是你完成大部分操作的地方。左侧的侧边栏有六个标签页，侧边栏底部还有一个小状态卡片，一眼即可看到当前连接和出口 IP。

标签页用途
连接仪表盘首页——连接或断开、查看实时吞吐量、切换服务器。
服务器添加、编辑、删除，以及选择当前使用的服务器。
日志实时活动日志。
流量统计你随时间传输了多少数据。
设置端口、启动、macOS 网络、菜单栏等——所有更改即时生效。
工具让 Claude Code 走此代理，以及一个用于移除所有代理的应急按钮。

连接标签页（仪表盘）

「连接」是首页，中心是一个大号电源圆环——点击它即可连接或断开。围绕它的是实时吞吐量和延迟图、一张用于快速切换服务器的卡片，以及三个一点即切的选项卡片。

下方的三个选项卡片一点即切——无需保存：

  • 路由 Mac 流量——把 Mac 的所有流量都通过隧道转发（见下文）。
  • 自动重连——隧道意外中断时悄悄重新连接。
  • 登录时启动——登录时自动启动 Tunnel Proxy。

菜单栏弹出面板

点击菜单栏中的盾牌图标即可打开一个紧凑面板——当你不想打开整个窗口时很方便。

弹出面板把要点浓缩在一小块空间里：

  • 状态卡——电源圆环（点击连接 / 断开）、你的出口 IP，以及当前延迟。
  • 速度和服务器卡——实时速度与当前服务器。点击 Switch ▾ 即可切换到另一台服务器。
  • 开关卡——路由 Mac 流量 和 自动重连，各一点即切。
  • 断开——停止隧道。
  • 打开应用 / 退出——打开主窗口，或退出 Tunnel Proxy。

迷你窗口

迷你窗口是一个始终置顶的小浮窗，让你在使用其他应用时也能看到状态和一个 停止 按钮。可从弹出面板状态卡中的画中画图标打开，或从菜单栏的 View → Mini Window（⇧⌘M）打开。

读懂状态图标

菜单栏图标的形状和颜色会随连接状态而变化。

图标状态含义
空心盾牌（灰点）未连接流量直接连接——隧道已关闭。
闪烁的空心盾牌（黄点）正在连接 / 正在重连正在建立隧道。
左半填充盾牌（绿点）已连接隧道与代理已启用，一切就绪。
带感叹号的盾牌（红点）出错出现问题。打开 日志 标签页查看原因。

首次设置：添加你的服务器

在连接之前，你需要先把 SSH 服务器信息告诉应用。

  1. 打开主窗口，在侧边栏中选择 服务器 标签页。
  2. 点击 ＋ 添加服务器。

填写以下信息：

字段填写内容
名称任意你喜欢的标签，如「家里的服务器」。可选。
主机服务器地址，如 example.com 或 203.0.113.10。
用户名你在服务器上的登录名，如 root。
端口SSH 端口。通常为 22——如无特别说明请保持不变。

选择一种 认证方式——即你如何证明身份：

方式何时使用填写内容
密码你用密码登录你的 SSH 密码
私钥文件你有一个密钥文件（如在 ~/.ssh 中）点击 选择… 选取密钥；若密钥有口令则一并填写
默认密钥 / 代理你的 Mac 已配置好用密钥连接无需填写——自动使用你现有的密钥

点击 保存。

你的密码很安全。 密码和密钥口令保存在 macOS 钥匙串 中——这是 macOS 用来保管你其他密码的同一个安全保险库，绝不会写入明文文件。

你可以添加多台服务器并在「连接」标签页或菜单栏弹出面板中切换。在 服务器 标签页里，点击某一行左侧的圆圈即可把它设为当前使用的服务器。

连接

  1. 打开 连接 标签页（或点击盾牌图标打开弹出面板）。
  2. 确认 服务器 卡片中已选中正确的服务器。
  3. 点击 电源圆环。

几秒钟后，状态会变为绿色 / 已连接，并显示 出口 IP。这个 IP 就是你服务器的地址——证明你的流量正通过隧道传输。

要停止，再次点击电源圆环（或在弹出面板中按 断开）。

日常选项

这些是 连接 标签页上一点即切的选项卡片（其中两个也会作为开关卡出现在弹出面板中）。它们一点即生效。

自动重连

建议保持开启。如果隧道意外中断——比如 Wi-Fi 抖动一下——应用会在后台悄悄为你重新连接。重连时你可能会短暂看到黄色的「正在重新连接…」状态。

路由 Mac 流量

  • 关闭（默认）： 只有你手动指向代理的应用才会走隧道。
  • 开启： 应用会修改你 Mac 的全局网络设置，让所有应用自动走隧道。

如果你希望 Mac 上的一切都走隧道、而不必逐个配置应用，就把它开启。断开或退出时，如果你想恢复直连，记得把它重新关闭。

登录时启动

开启后，每次登录 Mac 时 Tunnel Proxy 都会自动启动，始终守候在菜单栏中。

设置 标签页里还有一个 启动时自动连接 选项。两者都开启，隧道就会在你每次开机时自动建立。

显示网络速度 位于 设置 标签页的 菜单栏 分组中。开启后，菜单栏图标旁边会显示实时的上传/下载速度，方便你一眼看到流量在流动。

设置标签页

设置 标签页汇集了应用的所有选项，按卡片分组。这里没有保存按钮——所有更改即时生效。如果你输入了无效的值，相应卡片下方会出现橙色提示。

  • 端口——SOCKS 端口 和 HTTP 代理端口，即应用在本机监听的端口号（默认 1080 和 8118）。仅在与其他程序冲突时才需修改。
  • 启动——登录时启动 与 启动时自动连接（见上文）。
  • macOS——路由 Mac 流量 作用于哪个 网络服务（Wi-Fi、以太网……）、自动重连（守护进程） 开关，以及 守护进程间隔（秒）——自动重连检查的运行频率。
  • 菜单栏——显示菜单栏图标 和 显示网络速度。
  • 运行环境——确认应用所需的组件（privoxy 已内置；ssh 和 curl 由 macOS 自带）都已就绪。
  • 流量统计——记录流量统计 会在本地记录你随时间收发的数据量（见 流量统计）。它只记录数据大小——绝不记录你访问的网站——且数据仅保存在你的 Mac 上。清除统计… 会永久删除这些记录。
  • 文件——显示配置文件和日志文件的位置，并提供 在访达中显示 按钮。

工具标签页

工具 标签页提供一键将其他程序指向本应用 HTTP 代理的辅助功能。

  • 让 Claude Code 走此代理——开启后，Claude Code 的流量将通过隧道转发。它会把标准的 HTTP 代理环境变量（http_proxy、https_proxy 及其大写形式，以及针对 localhost 的 no_proxy）写入 ~/.claude/settings.json，指向 http://127.0.0.1:8118（或你在「设置」标签页设置的 HTTP 代理端口）。关闭 时仅移除这些键，Claude Code 的其他设置保持不变。修改后请重启 Claude Code 使其生效。
  • 设置文件——显示 ~/.claude/settings.json 的路径，并提供 Reveal 按钮。
  • 移除所有代理——一个应急按钮，会为当前的网络服务关闭所有系统代理（HTTP、HTTPS、SOCKS 以及自动/PAC 代理）。当代理设置意外残留——例如应用崩溃之后——导致 Mac 无法正常直连上网时，可用它来清理。此操作需要管理员密码。

如果之后修改了 HTTP 代理端口，请回到此处将 Claude Code 开关关闭再开启，以写入新的端口。

日志标签页

如果连接失败或行为异常，日志会准确显示发生了什么。

  1. 在侧边栏中打开 日志 标签页。
  2. 日志会实时输出：
    • 筛选…——输入关键字只显示匹配的行。
    • 自动滚动——始终显示最新的日志行。
    • 清除——清空显示内容。

日志按颜色区分：绿色表示成功，橙色表示警告，红色表示错误。

流量统计标签页

Tunnel Proxy 可以持续统计经隧道传输的数据量，让你了解自己随时间的使用情况。

在 设置 标签页里开启 记录流量统计 即可启用。它只记录数据量（上传和下载的字节数），不会记录你访问了哪些网站，所有数据都保存在你的 Mac 本地。

打开 流量统计 标签页即可查看：

  • 时间范围——在 今天、本周、本月、全部 之间切换。
  • 服务器筛选——将会话列表限定为某一台服务器，或显示全部。
  • 合计——所选范围内的 下载、上传 和 总计 数据。
  • 用量随时间变化——下载（蓝色）与上传（绿色）流量的柱状图，「今天」按小时、更长的范围按天显示。
  • 会话——每一次连接，包含其时间段、所用服务器以及传输的数据量。绿色圆点表示仍在进行中的会话。
  • 导出…——将图表数据保存为 .csv 文件，便于在电子表格中使用。

在你开启之前不会记录，因此早期的会话不会出现。底部的状态栏会显示是否正在记录以及自何时开始。可在 设置 标签页中使用 清除统计… 抹除历史记录。

常见问题

我还需要安装别的东西吗（Homebrew、其他工具）？
不需要。所有组件都已内置在应用中。ssh 和 curl 是 macOS 自带的。

这是 VPN 吗？
对你的流量来说效果类似，但它是连到你自己服务器的 SSH 隧道，而不是商业 VPN 服务。服务器由你自己提供。

我的密码存在哪里？
存在 macOS 钥匙串——系统的安全保险库里，绝不会以明文形式保存。

它会拖慢我的网速吗？
你的流量会经由服务器多一次中转，因此会有少量额外开销。速度主要取决于你服务器的网络连接。

如何彻底卸载？
先断开连接，关闭 登录时启动 与 路由 Mac 流量，然后退出应用，把 Tunnel Proxy 从「应用程序」拖入废纸篓即可。

想知道后台到底发生了什么？日志 标签页永远是排查问题的第一站。
